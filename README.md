# Gifts4U – Christmas E—commerce Website

A responsive front—end e—commerce website built using HTML, CSS, and JavaScript, designed to simulate a festive online shopping experience.

**Note**: This project is an improved version of my previous project (**Gifts4U—2**), with enhancements in design, structure, and user experience.

## Live Demo

https://springs2401.github.io/Gifts4U/

## Project Overview

This project focuses on applying key web design and development principles to create a functional and visually engaging user experience. It integrates visual design principles (colour theory, typography, hierarchy), responsive design techniques, and interactive features using JavaScript to simulate a basic e—commerce flow.

## Design Highlights

— Festive colour palette (red, green, gold) aligned with seasonal theme  
— Clear visual hierarchy using typography, spacing, and contrast  
— Consistent layout and styling across pages  
— Application of Gestalt principles such as proximity, similarity, and alignment  
— Structured layout to support readability and intuitive navigation

## Technologies Used

— **HTML5**: structure and content organisation  
— **CSS3**: layout, styling, Flexbox, and responsiveness  
— **JavaScript**: interactivity and dynamic behaviour

## Key Features

— Responsive design across multiple screen sizes  
— Product listing using structured grid layouts  
— Add to Cart functionality  
— Cart updates (quantity and total calculation)  
— Persistent cart using localStorage  
— Mobile navigation with hamburger menu  
— Sticky navigation bar for improved usability

## Accessibility (Basic Implementation)

— Descriptive alt text for images  
— Visible focus states for keyboard navigation  
— Basic ARIA labels for improved semantic clarity  

**Note**: Accessibility is partially implemented and can be improved further to fully meet WCAG standards.

## Limitations

— Not built using a mobile—first approach  
— Limited accessibility compliance  
— No backend integration (front—end only simulation)  
— No advanced optimisation techniques (e.g., lazy loading, minification)  
— File—based structure limits scalability  

## Future Improvements

— Adopt a mobile—first design strategy  
— Improve accessibility using semantic HTML and WCAG guidelines  
— Introduce component—based architecture (e.g., React)  
— Integrate backend for full e—commerce functionality  
— Apply performance optimisation techniques
